## Requirement R03 — True long-context test instead of only larger num_ctx

Requirement text:
The long-context test must feed a deterministic long prompt so `prompt_eval_count` can prove actual context usage. Merely setting `num_ctx=8192` is insufficient.

Changes per file:
- `ollama-test-RTX3090.sh`: add `--long-prompt-words N`; generate deterministic long prompt; store prompt chars/words; calculate context fill percentage.
- `ollama-test-and-monitor-RTX3090.sh`: pass `--long-prompt-words` to test script.
- `README.md`: document long-context test meaning and caveats.

Verification instruction:
- Run fake test and confirm payload `04_longctx_gpu.json` contains many generated words.
- Confirm `summary.csv` has `prompt_chars`, `prompt_words`, and `context_fill_pct` columns.

## Requirement R04 — Correct performance classification and non-misleading averages

Requirement text:
Summaries must not mix cold sanity, warm single-request, long-context, and concurrency rows into one headline average. Report cold load, warm single-request throughput, long-context throughput, and concurrency separately.

Changes per file:
- `ollama-test-RTX3090.sh`: add `category` column and category-aware summary logic.
- `ollama-test-and-monitor-RTX3090.sh`: use category-aware metrics in combined summary.
- `README.md`: explain headline metrics.

Verification instruction:
- Run fake workflow and confirm terminal summary contains `Warm`, `Cold`, `LongCtx`, and `Concurrency` lines.
- Confirm the old generic `GPU avg ... rows` headline is absent.

## Requirement R05 — Concurrency aggregate throughput

Requirement text:
Concurrency output must report aggregate system throughput across parallel requests, not only per-request API token/sec values.

Changes per file:
- `ollama-test-RTX3090.sh`: add `concurrency-aggregate.csv`; measure wall-clock duration for concurrent group; calculate aggregate eval tokens/sec.
- `ollama-test-and-monitor-RTX3090.sh`: include aggregate concurrency in combined summary.
- `README.md`: explain aggregate vs per-request throughput.

Verification instruction:
- Run fake workflow with `--concurrency 2`.
- Confirm `concurrency-aggregate.csv` exists and terminal summary includes aggregate concurrency throughput.

## Requirement R06 — Correct curl/timeout return-code capture

Requirement text:
