ollama-info v1.7.1 plan
=========================

Trigger
-------
Review of user-provided v1.7 console output and result archives from:
- ollama-test-RTX3090-20260611-132715.zip (qwen3.6:27b)
- ollama-test-and-monitor-RTX3090-20260611-133048.zip (qwen3.6:35b)

Findings to fix
---------------
1. The qwen3.6:35b run was reported as a clean "single GPU" visible-answer benchmark, but the nested test archive's ollama-ps-after.txt showed a mixed CPU/GPU residency: 15%/85% CPU/GPU. That is not a full-GPU resident benchmark.
2. The qwen3.6:35b run started while qwen3.6:27b was already resident. The run's load-state.txt incorrectly reported model_resident_before=absent and cold_verified=1 because it only checked the tested model, not other resident models. This makes FirstReqLoad a model-switch/eviction/offload observation, not verified cold load.
3. TTFT averages included first-load and long-context rows in a single aggregate. This obscured practical warm interactive latency.
4. Monitor reports did not classify Ollama residency/offload state from `ollama ps` snapshots.

Implementation direction
------------------------
Patch v1.7 to v1.7.1 with targeted fixes:
- Make observed-mode ColdVerified strict: observed mode does not claim verified cold.
- Detect other resident models before first request and emit `model_switch_observed`.
- Capture resident model count/list into load-state.txt.
- Classify post-run residency as `full_gpu`, `cpu_gpu_offload`, `cpu_resident`, or `unknown` from `ollama ps`.
- Replace terminal/orchestrator "single GPU" language with "single-request" unless full GPU residency is actually observed.
- Split latency reporting into FirstTTFT, WarmTTFT, and TTFTall.
- Add monitor residency/offload report section.

Out of scope for v1.7.1
-----------------------
No change to the main v1.7 benchmark profiles or prompt design. The attached result archives show generation-only, concurrency=1 runs; adding new agentic/concurrency workloads is reserved for the next feature release unless explicitly requested as a larger benchmark expansion.
