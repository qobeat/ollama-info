ollama-info v1.8 implementation plan
====================================

Goal
----
Produce v1.8 from v1.7.1 after reviewing the three new RTX 3090 test results. The release must remove default dependence on currently loaded Ollama models, add ADOS-oriented capability prompts, support multi-model commands, centralize wrapper logic, and maintain ADOS apply/verify evidence.

Source review inputs
--------------------
- ollama-test-RTX3090-20260611-141433.zip: qwen3.6:27b direct v1.7.1 run.
- ollama-test-RTX3090-20260611-141928.zip: qwen3.6:35b direct v1.7.1 run after qwen3.6:27b.
- ollama-test-and-monitor-RTX3090-20260611-142622.zip: qwen3.6:35b monitored v1.7.1 run while model was already resident.
- Pasted text.txt: console transcript for the same runs.
- requirements.improved.md: ADOS execution, evidence, materialization, self-evaluation, package hygiene, and AI-playground alignment authority.

Findings
--------
1. v1.7.1 fixed the serious v1.7 semantic errors: observed mode no longer claims verified cold, qwen3.6:35b offload is reported, FirstTTFT/WarmTTFT are separated, and stale "single GPU" wording was removed.
2. v1.7.1 still defaults to observed load semantics, so tests can depend on prior model residency unless the user explicitly selects another mode.
3. qwen3.6:27b direct run had full-GPU residency and valid long-context row but a very large first request path, so default empty-card verification is needed.
4. qwen3.6:35b direct run correctly reported CPU/GPU offload and model-switch effects, but the result remains not comparable with clean full-GPU qwen3.6:27b.
5. qwen3.6:35b monitored run showed warm/resident behavior and hardware warnings, but it did not represent an empty-card run.
6. The console showed a usability gap: invoking scripts relative to the wrong working directory failed; a top-level wrapper is needed.
7. Bashrc and script wrappers still duplicate routing logic; v1.8 should centralize command behavior in one wrapper.

Objectives
----------
OBJ-001 Add default empty-card load mode that unloads all resident Ollama models before testing.
OBJ-002 Add ADOS default generation profile with first, second, and third prompts for coding, essay, and internet-access boundary behavior.
OBJ-003 Preserve legacy performance benchmark rows behind --profile perf / --profile legacy-perf.
OBJ-004 Add scripts/ollama.sh as the canonical wrapper for bashrc functions and multi-model commands.
OBJ-005 Support commands like: ollama test qwen3.6:35b qwen3.6:27b.
OBJ-006 Convert ollama-bench-RTX3090.sh into a compatibility shim to remove routing-code duplication.
OBJ-007 Simplify bashrc to delegate to ollama.sh and preserve upstream Ollama pass-through.
OBJ-008 Add capability-analysis evidence generated from the three ADOS prompts.
OBJ-009 Update README, bashrc docs, changelog, manifest, and QA evidence.
OBJ-010 Verify syntax, route-only behavior, empty-card behavior, prompt payloads, package hygiene, and evidence parsing.

Implementation sequence
-----------------------
1. Preserve v1.7.1 as baseline.
2. Modify ollama-test-RTX3090.sh: version, profile, empty-card mode, all-model unload, ADOS prompts, capability-analysis evidence.
3. Modify ollama-test-and-monitor-RTX3090.sh: default empty-card, pass --profile, summary profile reporting.
4. Add scripts/ollama.sh with centralized status/start/stop/models/gpu/logs/test/bench/embed-test behavior and multi-model support.
5. Convert scripts/ollama-bench-RTX3090.sh to a compatibility shim.
6. Reduce bashrc/.bashrc to wrapper delegation.
7. Update documentation and QA artifacts.
8. Run deterministic fake Ollama/NVIDIA verification and package hygiene checks.
9. Package v1.8 delivery and quality evidence zips.
