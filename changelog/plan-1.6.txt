# ollama-info v1.6 implementation plan

## Goal

Ship a package update that distinguishes text-generation models from embedding-only models, implements a valid `/api/embed` benchmark path, and prevents RTX 3090/Ollama health summaries from implying that generation inference succeeded when the selected model cannot generate.

## Evidence reviewed

- `ollama test bge-m3` failed because every `/api/generate` request returned HTTP 400: `does not support generate`.
- `ollama test gemma3:1b` passed as a generation benchmark, proving the RTX 3090/Ollama generation path was operational.
- Direct `/api/embed` against `bge-m3:latest` returned one embedding vector with dimension 1024.
- Direct `/api/show` against `bge-m3:latest` returned `capabilities: ["embedding"]`, `architecture: bert`, `context_length: 8192`, and `embedding_length: 1024`.

## Atomic change plan

### R1 — Capability-aware preflight

Files:
- `scripts/ollama-common.sh`
- `scripts/ollama-test-RTX3090.sh`

Plan:
- Add helpers for `/api/show` and model-role classification.
- Store slim `/api/show` metadata by default.
- Classify embedding-only models before generation tests.

Success condition:
- `ollama test bge-m3` exits quickly with `unsupported_generate_for_embedding_model` rather than running four doomed `/api/generate` requests.

### R2 — Embedding benchmark mode

Files:
- `scripts/ollama-common.sh`
- `scripts/ollama-test-RTX3090.sh`
- `scripts/ollama-test-and-monitor-RTX3090.sh`
- `scripts/ollama-embed-test-RTX3090.sh`
- `bashrc/.bashrc`

Plan:
- Add `/api/embed` curl helper.
- Add direct test `--embedding` mode.
- Add monitored orchestrator `--embedding` pass-through.
- Add `ollama embed-test` Bash wrapper.
- Record vector count, vector dimension, endpoint, and embedding throughput fields in `summary.csv`.

Success condition:
- `ollama embed-test bge-m3` produces embedding rows and no generation-only false failure.

### R3 — Known failure class

Files:
- `scripts/ollama-test-RTX3090.sh`

Plan:
- Classify `does not support generate` as `unsupported_generate_for_embedding_model`.
- Add targeted likely cause and next action in `failure-hints.txt`.

Success condition:
- The final summary says the selected model is embedding-only and recommends `ollama embed-test` or a generation model.

### R4 — Summary wording fix

Files:
- `scripts/ollama-test-RTX3090.sh`
- `scripts/ollama-test-and-monitor-RTX3090.sh`

Plan:
- Separate telemetry collection from inference benchmark status.
- Print `Telemetry:` for monitor-derived hardware observations.
- Print `Inference:` for completed generation or embedding benchmark status.
- Make LongCtx/LongEmb N/A when the request failed before prompt evaluation.

Success condition:
- Failed generation preflight does not show a misleading completed-inference health pass.

### R5 — Role-aware model listing

Files:
- `scripts/ollama-common.sh`
- `scripts/ollama-status`
- `bashrc/.bashrc`

Plan:
- Make model command listings include role, size, and suggested command.
- Suggest `ollama embed-test` for embedding-only models.

Success condition:
- `ollama models` steers `bge-m3` to embedding mode.

### R6 — Archive bloat reduction

Files:
- `scripts/ollama-test-RTX3090.sh`

Plan:
- Store slim `/api/show` output by default.
- Add `--full-model-show` for verbose model metadata.

Success condition:
- Normal run archive remains compact while full metadata remains available on request.

### R7 — dmesg run-window split

Files:
- `scripts/ollama-test-RTX3090.sh`

Plan:

- Report `new during this run` separately from `historical since boot`.

Success condition:
- Old WSL/dxg/NVIDIA messages are not presented as new evidence for the current run.
